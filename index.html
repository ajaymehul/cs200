About ChocAn

Chocoholics Anonymous (ChocAn) is an organization dedicated to helping people addicted to chocolate in all its glorious forms. Members pay a monthly fee to ChocAn. For this fee they are entitled to unlimited consultations and treatments with health care professionals, namely, dietitians, internists, and exercise experts.

Glossary

Term Definition
Membership Card given to every member; includes their name, a nine-digit member number, and it incorporates a magnetic strip on which that information is encoded
Provider a healthcare professional who provides services to ChocAn members
Member person who pays fees to the ChocAn organization in exchange for services
Suspended a member is suspended if they have not paid membership dues for at least a month
Validated the member has paid their fees and is in good standing with ChocAn
Invalid Number no member is registered to the number entered into the ChocAn terminal, or the number is out of the range of the desired terminal specifications
Provider Directory an alphabetically ordered list of service names and corresponding service codes and fees
Date-of-Service needed in case technical difficulties prevent the provider from billing ChocAn at time of service
Service Code Unique 6-digit code that corresponds to the services that ChocAn could potentially be billed for. An error message is printed if the provider tries to enter a nonexistent code
Terminal each provider has a unique ChocAn computer terminal that connects to the ChocAn database, provides a way for members to pay their fees, and allows providers to collect fees
Main Accounting Procedure Run weekly at the ChocAn data center. Reads list of services provided that week and prints a report for each member and each provider
Electronic Funds Transfer (EFT) Data Data that is written to a disk to ensure that each provider is credited with the appropriate amount of funds
ChocAn Data Center ChocAn database that verifies member numbers and holds all of the known records of the members
Provider report received by each provider every week and includes the list of services he or she provided to ChocAn members
Member report a report given to each member that includes the list of services provided to that member during that week
Summary report given to the manager for accounts payable
Acme Accounting Services a third-party organization responsible for financial procedures

Main Use Case Details

User Validation

Use Case: User Validation
Context: Provider swipes a membership card from user to validate membership
Actors: Provider, Data Center
Main Success Scenario:
1. Member hands membership card to the provider.
2. The provider swipes the member card through card reader or enters the member number manually.
3. The software dials the ChocAn data center for validating the user.
4. If the data center finds the member valid, “Validated” appears on the one-line display. If not, the reason for invalid status is displayed ( “Invalid” or “Member suspended”) with fees owed (if applicable).

Billing

Use Case: Billing
Context: Provider must bill ChocAn after a service has been provided to a ChocAn member
Actors: Provider
Main Success Scenario:
1. The provider swipes the member card through card reader or enters the member number manually.
2. When the word ‘Validated’ appears, the provider keys in the date the service was provided.
3. The provider enters the service code (using the Provider Directory) that corresponds to the service provided.
4. The software product displays the name of the service corresponding to the code and asks the provider to verify that it is the correct service.
5. The provider verifies the service.
5.1. The provider can also enter comments about the service provided (optional).
6. The software product writes a record to disk that includes the current date and time, date service was provided, provider number, member number, service code, and comments.
7. The software product looks up the fee to be paid for that service and displays it.
8. The provider fills out a form that includes the current date and time, the date the service was provided, member name, member number, service code, and fee to be paid.
9. At the end of the week, the provider totals the fees to verify the amount to be paid to that provider by ChocAn for that week.
Extensions:
3a. The provider enters a nonexistent service code.
3a.1. An error message is printed.
3a.2. The use case continues at step 3.

Request Provider Directory

Use Case: Request Provider Directory

Actors: Provider
Main Success Scenario:
Gets the provider directory.
1. Reads the database and retrieves the contact information for the provided who requested the directory.
2. Emails the provider directory to the provider.

Account Management

Use Case: Account Management
Context: Each day operators can edit provider and member records.
Actors: Data Center, Operators
Main Success Scenario:
1. The ChocAn Data Center is run in interactive mode.

Generate Report

Use Case: Generate Report
Context: Report is generated and emailed to the respective person
Actors: Provider, Manager, Member
Main Success Scenario:
1. A report generated for the kind of actor (Member Report, Provider Report or Manager Report)
2. Reads the disk and retrieves contact information for the respective actor.
3. The generated report is sent to the actor using email (Member, Provider or Manager).

Provider Report

Use Case: Provider Report extends Generate Report
Context: Generates a Provider report
Actors:
Main Success Scenario:
1. Reads record of Providers who worked that week
2. Generates report for each provider with Provider name (25 characters), Provider number (9 digits), Provider street address (25 characters), Provider city (14 characters), Provider state (2 letters), Provider ZIP code (5 digits).
3. Reads record services provided by the specific provider.
4. Adds to the generated report information about services provided including Date of service (MM–DD–YYYY), Date and time data were received by the computer (MM–DD–YYYY HH:MM:SS), Member name (25 characters), Member number (9 digits), Service code (6 digits), Fee to be paid (up to $999.99), Total number of consultations with members (3 digits), Total fee for week (up to $99,999.99).

Member Report

Use Case: Member Report extends Generate Report
Context: Generates a Member report
Actors:
Main Success Scenario:
1. Reads record of members who availed services that week
2. Generates report for each member with Member name (25 characters), Member number (9 digits), Member street address (25 characters), Member city (14 characters), Member state (2 letters), Member ZIP code (5 digits).
3. Reads record services availed by the specific member.
4. Adds to the generated report information about services including Date of service (MM–DD–YYYY), Provider name (25 characters), Service name (20 characters).

Manager Report

Use Case: Manager Report extends Generate Report
Context: Generates a Manager summary report
Actors:
Main Success Scenario:
1. Reads record of manager and get's list of providers under the manager.
2. Reads EFT records of each provider under the Manager 3. Generates report with information for each provider to be paid that week, the number of consultations each had, and his or her total fee for that week.
4. The total number of providers who provided services, the total number of consultations, and the overall fee total are added to the report
